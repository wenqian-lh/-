2020.6-2020.7 博物馆订票和藏品浏览系统（独立开发）
开发环境： JDK 1.8/Oracle
开发工具：Eclipse+ PL/SQL Developer
项目描述： 基于C/S架构，采用JAVA进行开发，使用SWT工具包构建用户界面。前台用户界面包含：在线预约订票，数字文物库，博物馆活动，博物馆公告，博物院信息公开等。后台界面能够保证管理员对用户界面所有板块的信息进行管理。超级管理员对管理员的管理。
用户功能：
（1）	用户可以登录注册，修改个人信息。
（2）	订购十五天内的门票，以及退票。
（3）	在系统内浏览藏品图片，获取相关信息。
（4）	获取博物院所有的展览信息，发布的通知信息。
（5）	首页信息高度集成，操作方便，界面美观，且包含轮播图，诗词定时变化（一日一赏）等特色板块。
管理员功能：
（1）	管理员登录注册，超级管理员可对普通管理员的信息进行管理。
（2）	首页展示的信息管理（添加删除公告和诗词等）
（3）	藏品库信息管理。
（4）	展览信息管理。
（5）	对诗词库信息进行管理
技术要点：
1.	用户注册需要用户填写邮箱接收验证码，使用javax.mail发送带有验证码的邮件，以提高用户注册时的安全性。
2.	采用FillLayout充满式布布局加sashForm分割面板再结合StackLayout堆栈式布局的布局方式，不仅提高了程序运行效率且布局和谐，界面美观，操作方便，减少了用户的学习成本。
3.	利用Composite面板配合ScrolledComposite行成了滚动面板，以显示藏品，从而使用户可获取的信息更加丰富。 
4.	通过一个线程暂停和启动实现了轮播图和诗词轮换的效果。
项目总结：
1.	熟悉了SWT图形化界面的使用、布局方式、事件绑定，熟练了对Oracle数据库的“增删改查”操作。
2.	由于是第一次独立开发项目，在开发过程中经常不遵守命名规范，不写注释等问题，例如类名首字母要大写。通过和老师交流，多次指出这些问题，最终改正了这些问题，养成了良好的java开发习惯。
3.	在图形化界面的设计和布局上，由于刚开始各个功能模块没有确定下来，再加上没有相关经验，所以导致用户界面非常混乱，使用起来极不方便。最后通过不断摸索、改进、询问同学意见，最终设计出了一个既美观又实用的界面。使我明白了在项目开始之前明确各个模块之间的功能是多么重要，以及在程序编写过程中不断摸索改进，不断听取外界意见的重要性。
